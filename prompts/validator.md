# 人生 Skill 阀门验证器（Validator Gates）

任务：在正式写入与向用户交付前，必须通过由 A 到 D 的 4 道严格安全与质量测试闸门。如果碰到任意【硬性失败条件（Hard Fail）】，必须打回该轮生成，输出标准化修复清单。

## 验证闸门定义

### Gate A: 结构完整性（Structural Integrity）
**目标**：确保最终组装形态符合构建器双模板契约。
- ✅ 必须完全保留 `life_model.md` 与 `persona.md` 内要求的一级、二级标题结构。
- ✅ 若某一维度没有素材，必须标注 `[数据缺失：暂无本维度的特征收集]`，而非将该标题直接删除掉。
- ❌ **Hard Fail**：缺少任一模板规定的一级节点（如丢失了“风险与机会偏好”、“情绪反应靶点”）。

### Gate B: 证据合规性（Evidence Compliance）
**目标**：阻断幻觉，统一证据引用。
- ✅ 每条实际做出的特征或原则结论，必须在其尾部携带置信度，并且跟上统一证据协议 `[E序号|来源|时间]` 的追溯指针。
- ✅ 结论必须符合置信度阈值要求（高置信★★★必须能被至少3个证据支撑/追溯）。
- ❌ **Hard Fail**：出现了脱离素材分析发散出来的结论（结论没有挂载 E序号，或出现"似乎"、"大概"等无端揣测）。
- ❌ **Hard Fail**：遗漏标注置信度星标（★★★/★★/★）。

### Gate C: 逻辑一致性（Logical Consistency）
**目标**：保证双模型（Life 和 Persona）之间的统一与互信。
- ✅ 人物模型中的核心价值观/决策原则，不应与语气性格中的底层设定互斥。
- ✅ 新增合并记录（Merger阶段）如果没有经过用户确认就强行掩盖了 `[CONFLICT]` 冲突区域。
- ❌ **Hard Fail**：一边说“情绪外放易失控”，一边说“极度理智冷静防守”，且未提供场景隔离说明。

### Gate D: 隐私与伦理边界（Privacy & Ethical Boundaries）
**目标**：实现物理意义上的安全脱敏与合规。
- ✅ 所有出现过的第三方实体（人名、敏感地点、具体企业名）必须已被泛化替换或使用代号。
- ✅ 输出的模板中包含免责声明。
- ❌ **Hard Fail**：分析模型中保留了真实联系人真名（特别是未经授权的当事人）。
- ❌ **Hard Fail**：包含了涉及违法违规、自残或引导现实高危活动的决策路径。

---

## 验证执行与输出协议

### 流程触发
在所有的生成构建（Builder/Merger）完成后触发此 Validator 判断。

### 输出标准化验证报告

**场景 1：通过所有闸门**
```markdown
[System Log] Validator Check: PASS ✅
Gate A (结构): 100% 覆盖
Gate B (证据): 已检查 14 处结论，均绑定了合法 [E编号] 与 置信度。
Gate C (逻辑): 双文件结论逻辑锁定自洽。
Gate D (隐私): 未发现真名外溢。
=> 准许落盘交付。
```

**场景 2：触发闸门异常（返回给引擎/用户的修复清单）**
```markdown
[System Log] Validator Check: FAIL ❌

🚫 触发错误闸门：[Gate B - 证据合规性异常]
错误明细：【关系边界】模块下“对陌生人非常冷漠”未挂载证据序列号和置信度，疑似为模型自我发散。
修复建议要求：请返回数据池查找并挂载满足门槛要求的 `[E序号|来源|时间]`，或将其标记为 `[数据缺失]`。

🚫 触发错误闸门：[Gate D - 隐私边界异常]
错误明细：Persona.md 的常挂嘴边语录中出现了真实全名“张伟强”。
修复建议要求：立即脱敏并用泛用代号如“前公司老张”替换。

=> 构建挂起，强制打回修复。
```
